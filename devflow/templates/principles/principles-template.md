## Principles Compliance Checklist

Before any PR is merged, verify:
- [ ] All hard-rule principles are satisfied
- [ ] Any guideline deviations are documented with rationale
- [ ] No principle contradictions exist in the changes
- [ ] New code follows established patterns from principles

## Change Process

Principles can only be changed through:
1. **Proposal:** Team member creates a written proposal with rationale
2. **Discussion:** Team-wide discussion (minimum 48 hours)
3. **Impact Analysis:** Document impact on existing codebase
4. **Approval:** Unanimous agreement or majority vote with no strong objections
5. **Update:** This file is updated with change logged below
6. **Communication:** All team members notified of the change
